# 📘 ArtéMis 操作ガイド

このガイドは「日常運用で迷わない」ことを目的に、操作だけをまとめています。

## 1. 最初にやること

- サイドバーの `📥 Notionデータ取得` を押す
- 通信不安定時は `Driveデータスキップ機能ON` を使う

## 2. 新規登録（基本フロー）

1. `新規登録` で媒体を選択
2. タイトル/作者などで検索
3. 候補を `登録リストに追加`
4. `登録リスト` タブで必要項目を確認・修正
5. `一括登録` または単体登録

登録後はフォームが初期化され、次の登録に進めます。

## 3. 出演者管理（既存の出演に人を追加）

1. モードを `出演者管理` に変更
2. 対象の `出演` レコードを選択
3. 参加者を追加  
   - 演奏者マスタから追加  
   - 手入力で追加  
   - CSV一括取り込み（テンプレートDL→編集→取込）
4. `📥 この出演に参加者を登録`

同名（同出演）を再追加した場合は重複作成せず、既存行を更新します（upsert）。

## 4. 確認・修正ポイント

- 日本語タイトル/英語タイトルは登録前に編集可能
- `日本語タイトルを検索` ボタンで追加探索可能
- ロケーションは候補選択後に保存されます（テキスト入力のみでは保存されません）

## 5. データ管理（既存データの編集）

- `データ管理` で対象ページを展開
- `このページをリフレッシュ` で当該ページのみ再取得
- タイトル/媒体/ID/メタ情報/ロケーション/カバー画像を個別更新可能

## 6. 自動同期/リフレッシュ

- `自動同期`: 欠損中心の通常補填
- `リフレッシュ`: 強制再取得
- リフレッシュ後は設定に応じて整合修復を実行  
  - 手動（実行のみ）  
  - 半自動（高確度のみ）  
  - 自動（高確度＋重複整理）
- 実行条件は  
  - `出演/演奏曲を更新した時だけ`（推奨）  
  - `常に実行`  
  を選択可能

## 7. 失敗時の再実行

- リフレッシュ結果の `❌ 失敗・要確認` を展開
- 該当ページのIDや種別を調整して再実行
- 映画/ドラマはTMDB ID修正、アニメはAniList ID修正が基本

## 8. 出演・演奏曲リレーション運用

- Notionのリレーション `演奏曲` / `出演履歴` を作成済み前提
- 出演登録時に演奏曲DB・楽曲別担当者DBへ連携可能
- 出演者管理で演奏会参加者DBを追加・更新

## 9. Driveデータスキップ機能

- ON時はDrive保存/照合/一覧取得をスキップ
- 画面上では `⏭ スキップ中` と表示
- ネットワーク制限下のテストに有効

## 10. よくある確認項目

- Notion取得が遅い/止まる: ネットワーク制限またはAPI応答待ちの可能性
- Driveだけ失敗する: Drive API疎通/認証/フォルダ設定を確認
- 候補なし: 媒体とID種別（TMDB/AniList/IGDB/iTunes/ISBN）を再確認

## 11. 運用のコツ

- まず `媒体` を正しく選ぶ（誤媒体は誤画像の主因）
- 失敗ログから「そのページだけ」再実行する
- 大量件数時は分割運用＋個別再実行を併用する
